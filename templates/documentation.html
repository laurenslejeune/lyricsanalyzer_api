Lyrics analyzer API documentation

The lyrics analyer API allows for lexiconic analysis of provided lyrics. This documentation should allow the user to understand the functionality that it provides. All functions are built upon the Natural Language Toolkit, which provides functions for Natural Language processing.
All API data must be requested using GET requests. Currently, only English lyrics are supported.

Tokenization

Tokenization allows the user to turn an input text into separate "tokens". In this case, a list of these tokes will be returned to the user. In the current implementation, two kinds of tokenization are supported:

  • ../tokens/base/text
  • ../tokens/noStopwords/text

The base version simply turns every word into a token, while the noStopwords version removes all stop words from the text (a list of stop words can be found at this location).

Common words

Besides generating a list of tokens from a text, the API also supports a search for the most common words in a text. Four different requests are possible:

  • ../mostCommon/base/text/number
  • ../mostCommon/base/text
  • ../mostCommon/filtered/text/number
  • ../mostCommon/filtered/text

Besides a mandatory text, and optional parameter that can be added is the number of words that needs to be included. If no number is included, a default number of 10 is used instead.
While the base version simply uses all words in the text (very much like the base tokenizer includes all words), the filtered variant first removes stopwords and stems all words.

Text sentiment

It is possible to analyze a text to try and detect a sentiment in that text. More specifically, the VADER Sentiment Analysis model is used to do this. A get request providing a text at the following address will be analyzed:

../sentiment/vader/text

The postive (pos) and negative (neg) polarities can then be used to predict the sentiment of the text. Note that the analysis is lexicon-based, meaning that it will simply analyze the words used in the text. Unknown words increase the neutral (neu) score. This means the model is not a machine learning classification model.

Text analysis

If it is necessary to perform analysis in a single request, the following urls can be used:

  • ../analysis/text
  • ../analysis/text/number

This analysis function combines the outputs of the most common words and the VADER sentiment analysis. The number once again indicates the maximum amount of most commons words to return.
